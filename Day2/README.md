# Day 2 – Prompt Engineering
## ABTalks 60 Days Claude Challenge

### 📌 Objective
Learn the fundamentals of Prompt Engineering and understand how prompt quality directly impacts AI-generated outputs.

---

## 📝 Task

Compare the results of two different prompts:

### Prompt 1 (Basic)
> Create an image explaining Prompt Engineering.

### Prompt 2 (Engineered)
A detailed prompt that included:
- Beginner-friendly explanation
- Weak vs Improved Prompt examples
- LinkedIn-ready image requirements
- Claude-inspired color palette
- Professional and minimal design
- ABTalks 60 Days Claude Challenge branding

---

## 🔍 Observation

The first prompt generated a generic infographic because the AI had to make many decisions on its own.

The second prompt produced a much more targeted and professional result because it included:
- Clear objectives
- Audience context
- Design constraints
- Output format requirements

---

## 💡 Key Learning

Prompt Engineering is the practice of giving clear, specific, and structured instructions to AI.

The quality of the output often depends on the quality of the prompt.

**Better prompts = Better outputs**

---

## 🤯 Biggest Surprise

The AI itself did not change.

The results improved because the instructions became more detailed and intentional.

This showed how important clarity and context are when working with AI tools like Claude.

---

## 📊 Comparison

| Basic Prompt | Engineered Prompt |
|-------------|------------------|
| Generic request | Specific instructions |
| AI makes assumptions | AI follows guidance |
| Broad output | Targeted output |
| More revisions needed | Closer to final result |
| Less control | More control |

---

## 🎯 Takeaway

Prompt Engineering is not about using complex words.

It is about communicating clearly so AI understands exactly what you need.
