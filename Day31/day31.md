# Day 31 – AI Supply Chain Control Tower

## 🚀 Overview

For Day 31 of the **60 Days of Claude AI Challenge by ABTalksOnAI**, I built an **AI Supply Chain Control Tower** using Claude AI.

This project simulates the role of a **Head of Operations**, where users must monitor a live supply chain, respond to operational disruptions, and make critical decisions under time pressure. Instead of reading about supply chain management, users experience the challenges through an interactive real-time simulation.

---

## 🎯 Project Features

- 🚀 Interactive real-time supply chain simulation
- ⏱️ Three-minute operational challenge
- 📢 Live operational alerts with countdown timers
- ⚠️ Multiple alert priorities (Critical, High, Medium)
- 📊 Live KPI dashboard including:
  - Service Level
  - Customer Satisfaction
  - Inventory Health
  - Transport Efficiency
  - Operating Cost
  - Revenue Protected
- 🎮 Dynamic scoring system
- 📜 Event log tracking every operational decision
- ⏸️ Pause and Resume functionality
- ❓ Built-in Help Guide
- 📈 End-of-shift performance report
- 🏆 Letter grading system (A+ to D)
- 🔄 Replay with new scenarios

---

## 🧠 Key Learnings

### 1. AI can help build interactive simulations

Claude AI assisted in generating not only the user interface but also the underlying game logic, making it possible to create a complete simulation in significantly less time.

---

### 2. Real-world systems are event-driven

Unlike traditional applications, this project reacts continuously to changing events. Managing alerts, timers, and multiple concurrent operations helped me better understand event-driven programming.

---

### 3. Decision-making is about balancing trade-offs

Every operational decision affects multiple KPIs.

For example:

- Protecting customer satisfaction may increase operating costs.
- Ignoring an alert reduces service levels and revenue.
- Fast responses improve business performance but require careful prioritization.

There is rarely a single perfect solution.

---

### 4. Good UI improves decision making

Designing a clean dashboard with live metrics, countdown timers, and color-coded priorities demonstrated how visualization helps users react more effectively under pressure.

---

### 5. State management is critical

Managing timers, alerts, scoring, KPIs, and game progression highlighted the importance of maintaining application state throughout the simulation.
