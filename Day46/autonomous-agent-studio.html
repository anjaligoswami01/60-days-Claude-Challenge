◆How the stop-check governs the loop

Each round: Evaluator scores the current draft against the spec's correctness bar → Critic turns that score into concrete, fixable findings → Improver rewrites the draft using both → Memory Manager compresses the round into a running note fed forward. After Evaluator reports, three checks run in order every round: (1) plateau — gain < delta for 2 straight rounds, (2) threshold — score ≥ target, (3) hard cap — safety fallback only. The first one to fire ends the loop and branches execution to the Final Reviewer.

★Architecture overview

  • Planner — reads the raw spec once and produces a structured build plan (approach, data structures, edge cases) that anchors every later round.
  • Executor — turns the plan into the first working draft.
  • Evaluator — re-scores the draft against the spec every round; the only agent whose number literally drives the stop-check.
  • Critic — converts the score into a specific, prioritized punch-list the Improver can act on.
  • Improver — the only agent allowed to touch the code after round 0; always conditioned on the prior evaluation + critique.
  • Memory Manager — compresses history into a short running note so later rounds don't re-litigate settled ground.
  • Final Reviewer — runs once, after the loop exits, and independently signs off (or flags concerns) on the surviving draft.

★Extension ideas & further prompts

  • Add a Safety Monitor agent that screens every Improver output for unsafe patterns (eval, shell injection, unbounded recursion) before it's allowed to become the new draft.
  • Give the Executor a real sandboxed interpreter tool call so scores come from actual test execution, not just model judgment.
  • Let the Critic branch: "blocking" findings must be fixed next round; "polish" findings are deferred until threshold is close.
  • Persist Memory Manager notes across separate specs so the pipeline builds a house style over many runs.
  • Try prompting: "Design a 3-agent pipeline for legal-contract redlining with a human-approval stop condition instead of a score threshold."
  • Try prompting: "Extend this studio so Planner can request clarifying questions back to the user before Executor runs."
