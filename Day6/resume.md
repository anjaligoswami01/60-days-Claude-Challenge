
# Day 6/60 – AI Resume Optimizer by ABTalks

## Overview

This project was completed as part of the **60 Days Claude Challenge by ABTalks**.

The objective was to use AI to analyze an existing resume, identify ATS (Applicant Tracking System) weaknesses, optimize content and structure, and generate a recruiter-friendly one-page resume while maintaining complete accuracy and truthfulness.

---

## Objective

* Analyze an existing resume
* Evaluate ATS compatibility
* Identify formatting and keyword gaps
* Improve recruiter readability
* Create an ATS-friendly one-page resume
* Preserve all original achievements and experience

---

## Prompt Used

```text
ATS analysis for both resume

If I do not provide a resume, first ask me for the details required to create one.

Output EXACTLY two parts, nothing else:

PART 1 — ATS SCORE (keep short, no full report)

- Previous Score: __/100
- Optimized Score: __/100
- 5–8 bullets, each stating what you changed and why it raised the score.

FINAL RESUME

Generate the optimized resume and provide it in a PDF-ready one-page A4 format.

Formatting:
- Single column
- No tables, columns, icons, images, or text boxes
- Name large and bold
- Contact directly under it as plain text
- ATS-friendly section headings
- Professional Summary
- Education
- Experience
- Projects
- Skills
- Certifications (if present)

Rules:
- Use ONLY information from the resume.
- Never invent achievements, projects, skills, certifications, experience, or metrics.
- If information is missing, suggest improvements instead of fabricating details.
- Use strong action verbs.
- Remove redundancy.
- Keep everything truthful.
- Must fit on ONE A4 page.
- Optimize for ATS and recruiter readability.

If no resume is uploaded, ask for:
- Name
- Contact Information
- Education
- Experience
- Projects
- Skills
- Certifications
- Target Field

Then generate the resume.
```

---

## Process

1. Uploaded original resume
2. Performed ATS analysis
3. Evaluated keyword coverage
4. Identified formatting issues
5. Improved section hierarchy
6. Optimized resume content
7. Generated ATS-friendly one-page version
8. Compared ATS scores before and after optimization

---

## Results

| Metric              | Score     |
| ------------------- | --------- |
| Previous ATS Score  | 89/100    |
| Optimized ATS Score | 95/100    |
| Improvement         | +6 Points |

### Key Improvements

* ATS-friendly formatting
* Better keyword optimization
* Improved professional summary
* Cleaner section structure
* Enhanced recruiter readability
* Reduced redundancy
* Stronger achievement presentation
* Improved overall resume clarity

---

## Key Learnings

* ATS systems favor clean and structured resumes.
* Relevant keywords significantly improve screening performance.
* Quantified achievements create stronger impact.
* Simple resume layouts outperform complex designs.
* Recruiter readability is as important as ATS compatibility.
